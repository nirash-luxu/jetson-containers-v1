To connect a Node.js project to your local Ollama setup, we need to ensure the two Docker containers can communicate. Because you deployed Ollama using `network_mode: host`, the Ollama API is exposed directly to the Jetson's local network stack.

The easiest and most efficient way to achieve this on your Jetson Orin Nano is to deploy your Node.js container on that exact same host network. This avoids complex Docker bridge routing and allows Node.js to simply call `127.0.0.1:11434`.

Here is the complete guide and demo project. I have kept the code entirely free of internal comments, as you prefer, and provided detailed breakdowns below each file.

### Step 1: Create the Project Directory

```bash
mkdir -p ~/node_ollama_demo
cd ~/node_ollama_demo

```

These commands create an isolated folder for your Node.js application and navigate you inside it. Keeping projects in their own directories ensures the Docker build context only includes necessary files, which is crucial for saving space on your MicroSD card.

### Step 2: The Node Application Details

```bash
nano package.json

```

```json
{
  "name": "node-ollama-demo",
  "version": "1.0.0",
  "main": "index.js",
  "type": "module",
  "scripts": {
    "start": "node index.js"
  }
}
```

This `package.json` file defines your Node.js project. We are setting `"type": "module"` to allow the use of modern ES6 import syntax and native JavaScript features without needing extra compilation steps. We define the `start` script to execute your main application file.

```bash
nano index.js

```

```javascript
const OLLAMA_URL = "http://127.0.0.1:11434/api/generate";
const MODEL_NAME = "phi3";

async function askOllama(promptText) {
  try {
    console.log(`Sending prompt to Jetson GPU: "${promptText}"\n`);

    const response = await fetch(OLLAMA_URL, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        model: MODEL_NAME,
        prompt: promptText,
        stream: false,
      }),
    });

    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }

    const data = await response.json();
    console.log("OLLAMA RESPONSE:\n", data.response);
  } catch (error) {
    console.error("Communication Error:", error.message);
  }
}

askOllama("Explain quantum computing in one short sentence.");
```

This `index.js` file is your core application. It uses the native `fetch` API built into modern Node.js, meaning you do not need to install heavy external libraries like Axios. It points to `127.0.0.1` because the container will share the host network. The `stream: false` parameter forces Ollama to wait until the entire response is generated by the GPU before sending it back to your Node app, making it easier to log.

### Step 3: Containerizing the App

```bash
nano Dockerfile

```

```dockerfile
FROM node:20-alpine
WORKDIR /usr/src/app
COPY package.json ./
RUN npm install
COPY . .
CMD ["npm", "start"]

```

This `Dockerfile` packages your Node.js code into a container. We use the `node:20-alpine` base image. Alpine is a minimalist Linux distribution that creates incredibly small Docker images (often under 50MB), which is highly optimized for your MicroSD card storage limits. It sets the working directory, installs dependencies, copies your code, and triggers the start script.

### Step 4: The Deployment Configuration

```bash
nano docker-compose.yml

```

```yaml
services:
  node_app:
    build: .
    container_name: demo_node_client
    network_mode: host
    restart: "no"
```

This `docker-compose.yml` file tells Docker how to run your Node.js image. The `build: .` line instructs Docker to compile the image using the Dockerfile in the current directory. Setting `network_mode: host` is the critical bridge; it allows this container to share the Jetson's localhost alongside Ollama. We set `restart: "no"` because this is a one-off script that will execute, print the result, and finish.

### Step 5: Build and Run

To execute this demo, ensure your Ollama container is already running in the background, then run this command from inside your `~/node_ollama_demo` folder:

```bash
docker compose up --build

```

Docker will download the tiny Alpine Node image, build your project, and execute the script. You will see the prompt sent to the LLM, and seconds later, the generated response will print directly to your terminal.

---

To completely eradicate this Node.js project and recover every single kilobyte of space on your MicroSD card, we need to remove the container, the built images, the base images, the hidden build caches, and finally, the source code directory.

Run these commands in exact order.

### 1. Tear Down the Container and Built Image

```bash
cd ~/node_ollama_demo
docker compose down --rmi all

```

The first command ensures you are inside the project directory. The second command uses Docker Compose to stop the container and remove it. The `--rmi all` flag is the crucial addition here; it tells Docker to also delete the custom image it compiled specifically for your `demo_node_client`.

### 2. Remove the Base Node Image

```bash
docker rmi node:20-alpine

```

This command deletes the foundational Alpine Linux Node.js image that Docker downloaded from the internet to build your project.

### 3. Wipe the Hidden Build Cache

```bash
docker builder prune -a -f

```

When you ran the build command earlier, Docker saved intermediate cached layers of the installation process behind the scenes. This command forcefully purges Docker's entire internal build cache, guaranteeing no fragmented installation files are left occupying space on your disk.

### 4. Delete the Project Files

```bash
cd ~
rm -rf ~/node_ollama_demo

```

The first command moves you safely back to your home directory so you are not inside a folder you are about to delete. The second command forcefully and permanently recursively deletes the entire `node_ollama_demo` folder, taking the `index.js`, `package.json`, `Dockerfile`, and `docker-compose.yml` with it.

Your Jetson Orin Nano is now exactly as it was before we started the Node.js demo.
